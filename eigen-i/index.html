14. Eigenvalues and Eigenvectors

14.1Overview¶

Eigenvalues and eigenvectors are a relatively advanced topic in linear algebra.

At the same time, these concepts are extremely useful for

  • economic modeling (especially dynamics!)

  • statistics

  • some parts of applied mathematics

  • machine learning

  • and many other fields of science.

In this lecture we explain the basics of eigenvalues and eigenvectors and introduce the Neumann Series Lemma.

We assume in this lecture that students are familiar with matrices and understand the basics of matrix algebra.

We will use the following imports:

import matplotlib.pyplot as plt

14.2Matrices as transformations¶

Let’s start by discussing an important concept concerning matrices.

14.2.1Mapping vectors to vectors¶

One way to think about a matrix is as a rectangular collection of numbers.

Another way to think about a matrix is as a map (i.e., as a function) that transforms vectors to new vectors.

To understand the second point of view, suppose we multiply an n×mn \times mn×m matrix AAA with an m×1m \times 1m×1 column vector xxx to obtain an n×1n \times 1n×1 column vector yyy:

Ax=yAx = yAx=y
(1)

If we fix AAA and consider different choices of xxx, we can understand AAA as a map transforming xxx to AxAxAx.

Because AAA is n×mn \times mn×m, it transforms mmm-vectors to nnn-vectors.

We can write this formally as A ⁣:Rm→RnA \colon \mathbb{R}^m \rightarrow \mathbb{R}^nA:Rm→Rn.

You might argue that if AAA is a function then we should write A(x)=yA(x) = yA(x)=y rather than Ax=yAx = yAx=y but the second notation is more conventional.

14.2.2Square matrices¶

Let’s restrict our discussion to square matrices.

In the above discussion, this means that m=nm=nm=n and AAA maps Rn\mathbb R^nRn to itself.

This means AAA is an n×nn \times nn×n matrix that maps (or “transforms”) a vector xxx in Rn\mathbb{R}^nRn to a new vector y=Axy=Axy=Ax also in Rn\mathbb{R}^nRn.

Example 1
[21−11][13]=[52]\begin{bmatrix} 2 & 1 \\ -1 & 1 \end{bmatrix} \begin{bmatrix} 1 \\ 3 \end{bmatrix} = \begin{bmatrix} 5 \\ 2 \end{bmatrix}[2−1​11​][13​]=[52​]
(2)

Here, the matrix

A=[21−11]A = \begin{bmatrix} 2 & 1 \\ -1 & 1 \end{bmatrix}A=[2−1​11​]
(3)

transforms the vector x=[13]x = \begin{bmatrix} 1 \\ 3 \end{bmatrix}x=[13​] to the vector y=[52]y = \begin{bmatrix} 5 \\ 2 \end{bmatrix}y=[52​].

14.3Types of transformations¶

Let’s examine some standard transformations we can perform with matrices.

Below we visualize transformations by thinking of vectors as points instead of arrows.

We consider how a given matrix transforms

  • a grid of points and

  • a set of points located on the unit circle in R2\mathbb{R}^2R2.

To build the transformations we will use two functions, called grid_transform and circle_transform.

Each of these functions visualizes the actions of a given 2×22 \times 22×2 matrix AAA.

    r = min(1, 1-y/3)

14.3.1Scaling¶

A matrix of the form

[α00β]\begin{bmatrix} \alpha & 0 \\ 0 & \beta \end{bmatrix}[α0​0β​]
(4)

scales vectors across the x-axis by a factor α\alphaα and along the y-axis by a factor β\betaβ.

Here we illustrate a simple example where α=β=3\alpha = \beta = 3α=β=3.

A = np.array([[3, 0],  # scaling by 3 in both directions
              [0, 3]])
grid_transform(A)
circle_transform(A)

14.3.2Shearing¶

A “shear” matrix of the form

[1λ01]\begin{bmatrix} 1 & \lambda \\ 0 & 1 \end{bmatrix}[10​λ1​]
(5)

stretches vectors along the x-axis by an amount proportional to the y-coordinate of a point.

A = np.array([[1, 2],     # shear along x-axis
              [0, 1]])
grid_transform(A)
circle_transform(A)

14.3.3Rotation¶

A matrix of the form

[cos⁡θsin⁡θ−sin⁡θcos⁡θ]\begin{bmatrix} \cos \theta & \sin \theta \\ - \sin \theta & \cos \theta \end{bmatrix}[cosθ−sinθ​sinθcosθ​]
(6)

is called a rotation matrix.

This matrix rotates vectors clockwise by an angle θ\thetaθ.

θ = np.pi/4  # 45 degree clockwise rotation
A = np.array([[np.cos(θ), np.sin(θ)],
              [-np.sin(θ), np.cos(θ)]])
grid_transform(A)

14.3.4Permutation¶

The permutation matrix

[0110]\begin{bmatrix} 0 & 1 \\ 1 & 0 \end{bmatrix}[01​10​]
(7)

interchanges the coordinates of a vector.

A = np.column_stack([[0, 1], [1, 0]])
grid_transform(A)

More examples of common transition matrices can be found here.

14.4Matrix multiplication as composition¶

Since matrices act as functions that transform one vector to another, we can apply the concept of function composition to matrices as well.

14.4.1Linear compositions¶

Consider the two matrices

A=[01−10]andB=[1201]A = \begin{bmatrix} 0 & 1 \\ -1 & 0 \end{bmatrix} \quad \text{and} \quad B = \begin{bmatrix} 1 & 2 \\ 0 & 1 \end{bmatrix}A=[0−1​10​]andB=[10​21​]
(8)

What will the output be when we try to obtain ABxABxABx for some 2×12 \times 12×1 vector xxx?

[01−10]⏟A[1201]⏟B[13]⏞x→[01−1−2]⏟AB[13]⏞x→[3−7]⏞y\color{red}{\underbrace{ \color{black}{\begin{bmatrix} 0 & 1 \\ -1 & 0 \end{bmatrix}} }_{\textstyle A} } \color{red}{\underbrace{ \color{black}{\begin{bmatrix} 1 & 2 \\ 0 & 1 \end{bmatrix}} }_{\textstyle B}} \color{red}{\overbrace{ \color{black}{\begin{bmatrix} 1 \\ 3 \end{bmatrix}} }^{\textstyle x}} \rightarrow \color{red}{\underbrace{ \color{black}{\begin{bmatrix} 0 & 1 \\ -1 & -2 \end{bmatrix}} }_{\textstyle AB}} \color{red}{\overbrace{ \color{black}{\begin{bmatrix} 1 \\ 3 \end{bmatrix}} }^{\textstyle x}} \rightarrow \color{red}{\overbrace{ \color{black}{\begin{bmatrix} 3 \\ -7 \end{bmatrix}} }^{\textstyle y}}A[0−1​10​]​​B[10​21​]​​[13​]​x​→AB[0−1​1−2​]​​[13​]​x​→[3−7​]​y​
(9)
[01−10]⏟A[1201]⏟B[13]⏞x→[01−10]⏟A[73]⏞Bx→[3−7]⏞y\color{red}{\underbrace{ \color{black}{\begin{bmatrix} 0 & 1 \\ -1 & 0 \end{bmatrix}} }_{\textstyle A} } \color{red}{\underbrace{ \color{black}{\begin{bmatrix} 1 & 2 \\ 0 & 1 \end{bmatrix}} }_{\textstyle B}} \color{red}{\overbrace{ \color{black}{\begin{bmatrix} 1 \\ 3 \end{bmatrix}} }^{\textstyle x}} \rightarrow \color{red}{\underbrace{ \color{black}{\begin{bmatrix} 0 & 1 \\ -1 & 0 \end{bmatrix}} }_{\textstyle A}} \color{red}{\overbrace{ \color{black}{\begin{bmatrix} 7 \\ 3 \end{bmatrix}} }^{\textstyle Bx}} \rightarrow \color{red}{\overbrace{ \color{black}{\begin{bmatrix} 3 \\ -7 \end{bmatrix}} }^{\textstyle y}}A[0−1​10​]​​B[10​21​]​​[13​]​x​→A[0−1​10​]​​[73​]​Bx​→[3−7​]​y​
(10)

We can observe that applying the transformation ABABAB on the vector xxx is the same as first applying BBB on xxx and then applying AAA on the vector BxBxBx.

Thus the matrix product ABABAB is the composition of the matrix transformations AAA and BBB

This means first apply transformation BBB and then transformation AAA.

When we matrix multiply an n×mn \times mn×m matrix AAA with an m×km \times km×k matrix BBB the obtained matrix product is an n×kn \times kn×k matrix ABABAB.

Thus, if AAA and BBB are transformations such that A ⁣:Rm→RnA \colon \mathbb{R}^m \to \mathbb{R}^nA:Rm→Rn and B ⁣:Rk→RmB \colon \mathbb{R}^k \to \mathbb{R}^mB:Rk→Rm, then ABABAB transforms Rk\mathbb{R}^kRk to Rn\mathbb{R}^nRn.

Viewing matrix multiplication as composition of maps helps us understand why, under matrix multiplication, ABABAB is generally not equal to BABABA.

(After all, when we compose functions, the order usually matters.)

14.5Iterating on a fixed map¶

In economics (and especially in dynamic modeling), we are often interested in analyzing behavior where we repeatedly apply a fixed matrix.

For example, given a vector vvv and a matrix AAA, we are interested in studying the sequence

v,Av,AAv=A2v,…v, \quad Av, \quad AAv = A^2v, \quad \ldotsv,Av,AAv=A2v,…
(11)

Let’s first see examples of a sequence of iterates (Akv)k≥0(A^k v)_{k \geq 0}(Akv)k≥0​ under different maps AAA.

def plot_series(A, v, n):

    B = np.array([[1, -1],
                  [1, 0]])

    fig, ax = plt.subplots()

        ax.spines[spine].set_position('zero')

    r = 2.5

    y = r * np.sin(θ)

    y1 = y.reshape(1, -1)

Here with each iteration the vectors get shorter, i.e., move closer to the origin.

In this case, repeatedly multiplying a vector by AAA makes the vector “spiral in”.

We thus observe that the sequence (Akv)k≥0(A^kv)_{k \geq 0}(Akv)k≥0​ behaves differently depending on the map AAA itself.

We now discuss the property of A that determines this behavior.

14.6Eigenvalues¶

In this section we introduce the notions of eigenvalues and eigenvectors.

14.6.1Definitions¶

Let AAA be an n×nn \times nn×n square matrix.

If λ\lambdaλ is scalar and vvv is a non-zero nnn-vector such that

Av=λv.A v = \lambda v.Av=λv.
(12)

Then we say that λ\lambdaλ is an eigenvalue of AAA, and vvv is the corresponding eigenvector.

Thus, an eigenvector of AAA is a nonzero vector vvv such that when the map AAA is applied, vvv is merely scaled.

The next figure shows two eigenvectors (blue arrows) and their images under AAA (red arrows).

As expected, the image AvAvAv of each vvv is just a scaled version of the original

14.6.2Complex values¶

So far our definition of eigenvalues and eigenvectors seems straightforward.

There is one complication we haven’t mentioned yet:

When solving Av=λvAv = \lambda vAv=λv,

  • λ\lambdaλ is allowed to be a complex number and

  • vvv is allowed to be an nnn-vector of complex numbers.

We will see some examples below.

14.6.3Some mathematical details¶

We note some mathematical details for more advanced readers.

(Other readers can skip to the next section.)

The eigenvalue equation is equivalent to (A−λI)v=0(A - \lambda I) v = 0(A−λI)v=0.

This equation has a nonzero solution vvv only when the columns of A−λIA - \lambda IA−λI are linearly dependent.

This in turn is equivalent to stating the determinant is zero.

Hence, to find all eigenvalues, we can look for λ\lambdaλ such that the determinant of A−λIA - \lambda IA−λI is zero.

This problem can be expressed as one of solving for the roots of a polynomial in λ\lambdaλ of degree nnn.

This in turn implies the existence of nnn solutions in the complex plane, although some might be repeated.

14.6.4Facts¶

Some nice facts about the eigenvalues of a square matrix AAA are as follows:

  1. the determinant of AAA equals the product of the eigenvalues

  2. the trace of AAA (the sum of the elements on the principal diagonal) equals the sum of the eigenvalues

  3. if AAA is symmetric, then all of its eigenvalues are real

  4. if AAA is invertible and λ1,…,λn\lambda_1, \ldots, \lambda_nλ1​,…,λn​ are its eigenvalues, then the eigenvalues of A−1A^{-1}A−1 are 1/λ1,…,1/λn1/\lambda_1, \ldots, 1/\lambda_n1/λ1​,…,1/λn​.

A corollary of the last statement is that a matrix is invertible if and only if all its eigenvalues are nonzero.

14.6.5Computation¶

Using NumPy, we can solve for the eigenvalues and eigenvectors of a matrix as follows

from numpy.linalg import eig

A = ((1, 2),
     (2, 1))

A = np.array(A)
evals, evecs = eig(A)
evals  # eigenvalues
evecs  # eigenvectors

Note that the columns of evecs are the eigenvectors.

Since any scalar multiple of an eigenvector is an eigenvector with the same eigenvalue (which can be verified), the eig routine normalizes the length of each eigenvector to one.

The eigenvectors and eigenvalues of a map AAA determine how a vector vvv is transformed when we repeatedly multiply by AAA.

This is discussed further later.

14.7The Neumann Series Lemma¶

In this section we present a famous result about series of matrices that has many applications in economics.

14.7.1Scalar series¶

Here’s a fundamental result about series:

If aaa is a number and ∣a∣<1|a| < 1∣a∣<1, then

∑k=0∞ak=11−a=(1−a)−1 \sum_{k=0}^{\infty} a^k =\frac{1}{1-a} = (1 - a)^{-1}k=0∑∞​ak=1−a1​=(1−a)−1
(13)

For a one-dimensional linear equation x=ax+bx = ax + bx=ax+b where x is unknown we can thus conclude that the solution x∗x^{*}x∗ is given by:

x∗=b1−a=∑k=0∞akbx^{*} = \frac{b}{1-a} = \sum_{k=0}^{\infty} a^k bx∗=1−ab​=k=0∑∞​akb
(14)

14.7.2Matrix series¶

A generalization of this idea exists in the matrix setting.

Consider the system of equations x=Ax+bx = Ax + bx=Ax+b where AAA is an n×nn \times nn×n square matrix and xxx and bbb are both column vectors in Rn\mathbb{R}^nRn.

Using matrix algebra we can conclude that the solution to this system of equations will be given by:

x∗=(I−A)−1b x^{*} = (I-A)^{-1}bx∗=(I−A)−1b
(15)

What guarantees the existence of a unique vector x∗x^{*}x∗ that satisfies (15)?

The following is a fundamental result in functional analysis that generalizes (13) to a multivariate case.

prf:Theorem - Unknown Directive
:label: neumann_series_lemma

Let $A$ be a square matrix and let $A^k$ be the $k$-th power of $A$.

Let $r(A)$ be the **spectral radius** of $A$, defined as $\max_i |\lambda_i|$, where

* $\{\lambda_i\}_i$ is the set of eigenvalues of $A$ and
* $|\lambda_i|$ is the modulus of the complex number $\lambda_i$

Neumann's Theorem states the following: If $r(A) < 1$, then $I - A$ is invertible, and

$$
(I - A)^{-1} = \sum_{k=0}^{\infty} A^k
$$

We can see the Neumann Series Lemma in action in the following example.

A = np.array([[0.4, 0.1],
              [0.7, 0.2]])

evals, evecs = eig(A)   # finding eigenvalues and eigenvectors

r = max(abs(λ) for λ in evals)    # compute spectral radius
print(r)

The spectral radius r(A)r(A)r(A) obtained is less than 1.

Thus, we can apply the Neumann Series Lemma to find (I−A)−1(I-A)^{-1}(I−A)−1.

I = np.identity(2)  # 2 x 2 identity matrix
B = I - A
B_inverse = np.linalg.inv(B)  # direct inverse method
A_sum = np.zeros((2, 2))  # power series sum of A
A_power = I
for i in range(50):
    A_sum += A_power
    A_power = A_power @ A

Let’s check equality between the sum and the inverse methods.

np.allclose(A_sum, B_inverse)

Although we truncate the infinite sum at k=50k = 50k=50, both methods give us the same result which illustrates the result of the Neumann Series Lemma.

# Define a matrix A
A = np.array([[1, 0, 3],
              [0, 2, 0],
              [3, 0, 1]])

num_iters = 20

# Define a random starting vector b
b = np.random.rand(A.shape[1])

# Get the leading eigenvector of matrix A
eigenvector = np.linalg.eig(A)[1][:, 0]

errors = []
res = []

# Power iteration loop
for i in range(num_iters):
    # Multiply b by A
    b = A @ b
    # Normalize b
    b = b / np.linalg.norm(b)
    # Append b to the list of eigenvector approximations
    res.append(b)
    err = np.linalg.norm(np.array(b)
                         - eigenvector)
    errors.append(err)

greatest_eigenvalue = np.dot(A @ b, b) / np.dot(b, b)
print(f'The approximated greatest absolute eigenvalue is \
        {greatest_eigenvalue:.2f}')
print('The real eigenvalue is', np.linalg.eig(A)[0])

# Plot the eigenvector approximations for each iteration
plt.figure(figsize=(10, 6))
plt.xlabel('iterations')
plt.ylabel('error')
_ = plt.plot(errors)

class Arrow3D(FancyArrowPatch):
    def __init__(self, xs, ys, zs, *args, **kwargs):
        super().__init__((0, 0), (0, 0), *args, **kwargs)
        self._verts3d = xs, ys, zs

    def do_3d_projection(self):
        xs3d, ys3d, zs3d = self._verts3d
        xs, ys, zs = proj3d.proj_transform(xs3d, ys3d, zs3d,
                                           self.axes.M)
        self.set_positions((0.1*xs[0], 0.1*ys[0]),
                           (0.1*xs[1], 0.1*ys[1]))

        return np.min(zs)

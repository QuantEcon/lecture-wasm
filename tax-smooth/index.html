10. Tax Smoothing

10.1Overview¶

This is a sister lecture to our lecture on consumption-smoothing.

By renaming variables, we obtain a version of a model “tax-smoothing model” that Robert Barro Barro (1979) used to explain why governments sometimes choose not to balance their budgets every period but instead use issue debt to smooth tax rates over time.

The government chooses a tax collection path that minimizes the present value of its costs of raising revenue.

The government minimizes those costs by smoothing tax collections over time and by issuing government debt during temporary surges in government expenditures.

The present value of government expenditures is at the core of the tax-smoothing model, so we’ll again use formulas presented in present value formulas.

We’ll again use the matrix multiplication and matrix inversion tools that we used in present value formulas.

10.2Analysis¶

As usual, we’ll start by importing some Python modules.

A government exists at times t=0,1,…,St=0, 1, \ldots, St=0,1,…,S and faces an exogenous stream of expenditures {Gt}t=0S\{G_t\}_{t=0}^S{Gt​}t=0S​.

It chooses chooses a stream of tax collections {Tt}t=0S\{T_t\}_{t=0}^S{Tt​}t=0S​.

The model takes a government expenditure stream as an “exogenous” input that is somehow determined outside the model.

The government faces a gross interest rate of R>1R >1R>1 that is constant over time.

The government can borrow or lend at interest rate RRR, subject to some limits on the amount of debt that it can issue that we’ll describe below.

Let

  • S≥2S \geq 2S≥2 be a positive integer that constitutes a time-horizon.

  • G={Gt}t=0SG = \{G_t\}_{t=0}^SG={Gt​}t=0S​ be a sequence of government expenditures.

  • B={Bt}t=0S+1B = \{B_t\}_{t=0}^{S+1}B={Bt​}t=0S+1​ be a sequence of government debt.

  • T={Tt}t=0ST = \{T_t\}_{t=0}^ST={Tt​}t=0S​ be a sequence of tax collections.

  • R≥1R \geq 1R≥1 be a fixed gross one period interest rate.

  • β∈(0,1)\beta \in (0,1)β∈(0,1) be a fixed discount factor.

  • B0B_0B0​ be a given initial level of government debt

  • BS+1≥0B_{S+1} \geq 0BS+1​≥0 be a terminal condition.

The sequence of government debt BBB is to be determined by the model.

We require it to satisfy two boundary conditions:

  • it must equal an exogenous value B0B_0B0​ at time 0

  • it must equal or exceed an exogenous value BS+1B_{S+1}BS+1​ at time S+1S+1S+1.

The terminal condition BS+1≥0B_{S+1} \geq 0BS+1​≥0 requires that the government not end up with negative assets.

(This no-Ponzi condition ensures that the government ultimately pays off its debts -- it can’t simply roll them over indefinitely.)

The government faces a sequence of budget constraints that constrain sequences (G,T,B)(G, T, B)(G,T,B)

Bt+1=R(Bt+Gt−Tt),t=0,1,…SB_{t+1} = R (B_t + G_t - T_t), \quad t =0, 1, \ldots SBt+1​=R(Bt​+Gt​−Tt​),t=0,1,…S
(1)

Equations (1) constitute S+1S+1S+1 such budget constraints, one for each t=0,1,…,St=0, 1, \ldots, St=0,1,…,S.

Given a sequence GGG of government expenditures, a large set of pairs (B,T)(B, T)(B,T) of (government debt, tax collections) sequences satisfy the sequence of budget constraints (1).

The model follows the following logical flow:

  • start with an exogenous government expenditure sequence GGG, an initial government debt B0B_0B0​, and a candidate tax collection path TTT.

  • use the system of equations (1) for t=0,…,St=0, \ldots, St=0,…,S to compute a path BBB of government debt

  • verify that BS+1B_{S+1}BS+1​ satisfies the terminal debt constraint BS+1≥0B_{S+1} \geq 0BS+1​≥0.

    • If it does, declare that the candidate path is budget feasible.

    • if the candidate tax path is not budget feasible, propose a different tax path and start over

Below, we’ll describe how to execute these steps using linear algebra -- matrix inversion and multiplication.

The above procedure seems like a sensible way to find “budget-feasible” tax paths TTT, i.e., paths that are consistent with the exogenous government expenditure stream GGG, the initial debt level B0B_0B0​, and the terminal debt level BS+1B_{S+1}BS+1​.

In general, there are many budget feasible tax paths TTT.

Among all budget-feasible tax paths, which one should a government choose?

To answer this question, we assess alternative budget feasible tax paths TTT using the following cost functional:

L=−∑t=0Sβt(g1Tt−g22Tt2)L = - \sum_{t=0}^S \beta^t (g_1 T_t - \frac{g_2}{2} T_t^2 )L=−t=0∑S​βt(g1​Tt​−2g2​​Tt2​)
(2)

where g1>0,g2>0g_1 > 0, g_2 > 0g1​>0,g2​>0.

This is called the “present value of revenue-raising costs” in Barro (1979).

The quadratic term −g22Tt2-\frac{g_2}{2} T_t^2−2g2​​Tt2​ captures increasing marginal costs of taxation, implying that tax distortions rise more than proportionally with tax rates.

This creates an incentive for tax smoothing.

Indeed, we shall see that when βR=1\beta R = 1βR=1, criterion (2) leads to smoother tax paths.

By smoother we mean tax rates that are as close as possible to being constant over time.

The preference for smooth tax paths that is built into the model gives it the name “tax-smoothing model”.

Or equivalently, we can transform this into the same problem as in the consumption-smoothing lecture by maximizing the welfare criterion:

W=∑t=0Sβt(g1Tt−g22Tt2)W = \sum_{t=0}^S \beta^t (g_1 T_t - \frac{g_2}{2} T_t^2 )W=t=0∑S​βt(g1​Tt​−2g2​​Tt2​)
(3)

Let’s dive in and do some calculations that will help us understand how the model works.

Here we use default parameters R=1.05R = 1.05R=1.05, g1=1g_1 = 1g1​=1, g2=1/2g_2 = 1/2g2​=1/2, and S=65S = 65S=65.

We create a Python namedtuple to store these parameters with default values.

TaxSmoothing = namedtuple("TaxSmoothing", 
                        ["R", "g1", "g2", "β_seq", "S"])

def create_tax_smoothing_model(R=1.01, g1=1, g2=1/2, S=65):
    """
    Creates an instance of the tax smoothing model.
    """
    β = 1/R
    β_seq = np.array([β**i for i in range(S+1)])

    return TaxSmoothing(R, g1, g2, β_seq, S)

10.3Barro tax-smoothing model¶

A key object is the present value of government expenditures at time 0:

h0≡∑t=0SR−tGt=[1R−1⋯R−S][G0G1⋮GS]h_0 \equiv \sum_{t=0}^S R^{-t} G_t = \begin{bmatrix} 1 & R^{-1} & \cdots & R^{-S} \end{bmatrix} \begin{bmatrix} G_0 \cr G_1 \cr \vdots \cr G_S \end{bmatrix}h0​≡t=0∑S​R−tGt​=[1​R−1​⋯​R−S​]⎣⎡​G0​G1​⋮GS​​⎦⎤​
(4)

This sum represents the present value of all future government expenditures that must be financed.

Formally it resembles the present value calculations we saw in this QuantEcon lecture present values.

This present value calculation is crucial for determining the government’s total financing needs.

By iterating on equation (1) and imposing the terminal condition

BS+1=0,B_{S+1} = 0,BS+1​=0,
(5)

it is possible to convert a sequence of budget constraints (1) into a single intertemporal constraint

∑t=0SR−tTt=B0+h0.\sum_{t=0}^S R^{-t} T_t = B_0 + h_0.t=0∑S​R−tTt​=B0​+h0​.
(6)

Equation (6) says that the present value of tax collections must equal the sum of initial debt and the present value of government expenditures.

When βR=1\beta R = 1βR=1, it is optimal for a government to smooth taxes by setting

Tt=T0t=0,1,…,ST_t = T_0 \quad t =0, 1, \ldots, STt​=T0​t=0,1,…,S
(7)

(Later we’ll present a “variational argument” that shows that this constant path minimizes criterion (2) and maximizes (3) when βR=1\beta R =1βR=1.)

In this case, we can use the intertemporal budget constraint to write

Tt=T0=(∑t=0SR−t)−1(B0+h0),t=0,1,…,S.T_t = T_0 = \left(\sum_{t=0}^S R^{-t}\right)^{-1} (B_0 + h_0), \quad t= 0, 1, \ldots, S.Tt​=T0​=(t=0∑S​R−t)−1(B0​+h0​),t=0,1,…,S.
(8)

Equation (8) is the tax-smoothing model in a nutshell.

Because we have built into our calculations that the government must satisfy its intertemporal budget constraint and end with zero debt, just barely satisfying the terminal condition that BS+1≥0B_{S+1} \geq 0BS+1​≥0, it should turn out that

BS+1=0.B_{S+1} = 0.BS+1​=0.
(13)

Let’s verify this with Python code.

First we implement the model with compute_optimal

def compute_optimal(model, B0, G_seq):

    R, S = model.R, model.S

    # present value of government expenditures
    h0 = model.β_seq @ G_seq     # since β = 1/R

    # optimal constant tax rate
    T0 = (1 - 1/R) / (1 - (1/R)**(S+1)) * (B0 + h0)
    T_seq = T0*np.ones(S+1)

    A = np.diag(-R*np.ones(S), k=-1) + np.eye(S+1)
    b = G_seq - T_seq
    b[0] = b[0] + B0
    B_seq = np.linalg.inv(A) @ b
    B_seq = np.concatenate([[B0], B_seq])

    return T_seq, B_seq, h0

def cost(model, T_seq):
    β_seq, g1, g2 = model.β_seq, model.g1, model.g2
    cost_seq = g1 * T_seq - g2/2 * T_seq**2

    return - cost(model, T_seq)

10.4.5Feasible Tax Variations¶

We promised to justify our claim that a constant tax rate Tt=T0T_t = T_0Tt​=T0​ for all ttt is optimal.

Let’s do that now.

The approach we’ll take is an elementary example of the “calculus of variations”.

Let’s dive in and see what the key idea is.

To explore what types of tax paths are cost-minimizing / welfare-improving, we shall create an admissible tax path variation sequence {vt}t=0S\{v_t\}_{t=0}^S{vt​}t=0S​ that satisfies

∑t=0SR−tvt=0.\sum_{t=0}^S R^{-t} v_t = 0.t=0∑S​R−tvt​=0.
(14)

This equation says that the present value of admissible tax path variations must be zero.

So once again, we encounter a formula for the present value:

  • we require that the present value of tax path variations be zero to maintain budget balance.

Here we’ll restrict ourselves to a two-parameter class of admissible tax path variations of the form

vt=ξ1ϕt−ξ0.v_t = \xi_1 \phi^t - \xi_0.vt​=ξ1​ϕt−ξ0​.
(15)

We say two and not three-parameter class because ξ0\xi_0ξ0​ will be a function of (ϕ,ξ1;R)(\phi, \xi_1; R)(ϕ,ξ1​;R) that guarantees that the variation sequence is feasible.

Let’s compute that function.

We require

∑t=0SR−t[ξ1ϕt−ξ0]=0\sum_{t=0}^S R^{-t}\left[ \xi_1 \phi^t - \xi_0 \right] = 0t=0∑S​R−t[ξ1​ϕt−ξ0​]=0
(16)

which implies that

ξ1∑t=0SϕtR−t−ξ0∑t=0SR−t=0\xi_1 \sum_{t=0}^S \phi_t R^{-t} - \xi_0 \sum_{t=0}^S R^{-t} = 0ξ1​t=0∑S​ϕt​R−t−ξ0​t=0∑S​R−t=0
(17)

which implies that

ξ11−(ϕR−1)S+11−ϕR−1−ξ01−R−(S+1)1−R−1=0\xi_1 \frac{1 - (\phi R^{-1})^{S+1}}{1 - \phi R^{-1}} - \xi_0 \frac{1 - R^{-(S+1)}}{1-R^{-1} } =0ξ1​1−ϕR−11−(ϕR−1)S+1​−ξ0​1−R−11−R−(S+1)​=0
(18)

which implies that

ξ0=ξ0(ϕ,ξ1;R)=ξ1(1−R−11−R−(S+1))(1−(ϕR−1)S+11−ϕR−1)\xi_0 = \xi_0(\phi, \xi_1; R) = \xi_1 \left(\frac{1 - R^{-1}}{1 - R^{-(S+1)}}\right) \left(\frac{1 - (\phi R^{-1})^{S+1}}{1 - \phi R^{-1}}\right)ξ0​=ξ0​(ϕ,ξ1​;R)=ξ1​(1−R−(S+1)1−R−1​)(1−ϕR−11−(ϕR−1)S+1​)

Key Idea: if ToT^oTo is a budget-feasible tax path, then so is To+vT^o + vTo+v, where vvv is a budget-feasible variation.

Given RRR, we thus have a two parameter class of budget feasible variations vvv that we can use to compute alternative tax paths, then evaluate their welfare costs.

Now let’s compute and plot tax path variations

def compute_variation(model, ξ1, ϕ, B0, G_seq, verbose=1):
    R, S, β_seq = model.R, model.S, model.β_seq

    ξ0 = ξ1*((1 - 1/R) / (1 - (1/R)**(S+1))) * ((1 - (ϕ/R)**(S+1)) / (1 - ϕ/R))
    v_seq = np.array([(ξ1*ϕ**t - ξ0) for t in range(S+1)])
    
    if verbose == 1:
        print('check feasible:', np.isclose(β_seq @ v_seq, 0))     

    T_opt, _, _ = compute_optimal(model, B0, G_seq)
    Tvar_seq = T_opt + v_seq

    return Tvar_seq
